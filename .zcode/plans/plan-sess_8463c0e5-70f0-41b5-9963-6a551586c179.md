# SeedNet ‚Äî Implementation Plan

## ‚öÝÔ∏è One necessary deviation from the spec (needs your awareness)

The spec lists `bittorrent-dht` as a crate, but **`bittorrent-dht` no longer exists on crates.io** ‚Äî confirmed via `cargo info bittorrent-dht` ‚Üí *"could not find `bittorrent-dht`"*. It was renamed/evolved into **`mainline` v7.0.0** (same maintainer lineage: `pubky`/`nuhvi`, repo `github.com/pubky/mainline`). `dht` v6.4.0 is an older alias. SeedNet will use **`mainline` 7.0.0** ‚Äî it provides the exact BEP_0005 Mainline DHT `announce`/`get`/`lookup` functionality the spec requires, is fully async, and is actively maintained. SeedNet still uses DHT *only* for peer discovery ‚Äî no torrent/piece/tracker logic.

All other spec-named crates resolve cleanly (snow 0.10.0, x25519-dalek 3.0, ed25519-dalek 3.0, blake3, sha1 0.11, hkdf 0.13, tun 0.8.13, socket2 0.6, quinn 0.11, etc.).

## Identity model (per your decision: per-device key + seed prologue)

```
passphrase ("correct horse battery staple")
   ‚îÇ
   HKDF-SHA256(salt = b"seednet-v1", info = b"network-secret", len = 32)
   ‚îÇ
   ‚ñº
NetworkSecret (32 bytes, identical on all devices)
   ‚îú‚îÄ‚îÄ SHA1 ‚îÄ‚îÄ‚ñ∂ infohash (20 bytes)         ‚Üê DHT announce/lookup key
   ‚îî‚îÄ‚îÄ Noise prologue (32 bytes)             ‚Üê gates who may join the network

Per-device (generated ONCE on first run, persisted to ~/.seednet/identity.bin):
   random 32-byte device seed
   ‚îú‚îÄ‚îÄ Ed25519 keypair  (identity / signing)
   ‚îî‚îÄ‚îÄ X25519 static key (Noise XX static key)
   ‚îî‚îÄ‚îÄ OverlayIP = 10.88.<h(pubkey) mod 254 + 1>.<h2(pubkey) mod 256>  ‚Üê deterministic, low-collision
```

This matches Tailscale/Nebula semantics: shared network secret authenticates *membership*, distinct per-device keys give distinct identities, distinct overlay IPs, and proper forward secrecy.

## Milestone-by-milestone deliverables

Every milestone: **compiles clean** (`cargo build --workspace`), **tests pass** (`cargo test --workspace`), **runs** (`cargo run -p seednet-cli ...`), documented verification steps in README. No `TODO`, no stubs, no pseudocode.

### Milestone 1 ‚Äî CLI + Identity from Seed
- `common`: `Seed` (newtype wrapping passphrase bytes), `NetworkSecret`([u8;32]), `PeerId`, `OverlayAddr`, constants (`DEFAULT_PORT 4242`, `OVERLAY_SUBNET 10.88.0.0/16`, `MTU 1280`).
- `crypto`: `derive_network_secret(&Seed) -> NetworkSecret` (HKDF-SHA256), `infohash(&NetworkSecret) -> [u8;20]` (SHA-1), `DeviceKeys { ed25519, x25519 }` generation, `overlay_ip(&Ed25519Pubkey) -> Ipv4Addr`.
- `config`: `Identity` struct, load/save to `~/.seednet/identity.bin` (CBOR or postcard), `StateDir` helper.
- `cli`: clap with subcommands `up <seed>`, `down`, `status`, `identity <seed>`.
- **Verification**: `cargo run -p seednet-cli -- identity "correct horse battery staple"` prints infohash + (placeholder) identity. Two runs ‚Üí same infohash. Unit tests: deterministic derivation, key round-trip.

### Milestone 2 ‚Äî DHT announce/lookup
- `dht` crate: `DhtDiscovery { async_dht: AsyncDht }`, `DhtDiscovery::start(port)`, `announce(&infohash, listen_port)`, `lookup(&infohash) -> Vec<PeerAddr>` (PeerAddr = SocketAddr + peer hint).
- Use `mainline::async_dht::AsyncDht::builder()...build()`, `dht.announce(infohash, port)`, `dht.get(&Id::from(infohash))` collecting `Peer` results.
- `cli`: add `discover <seed>` command that boots DHT, announces, looks up, prints discovered peers for ~30s.
- **Verification**: `cargo run -- discover "<seed>"` joins Mainline, prints announce + any peers. Unit test: infohash ‚Üî SHA-1 roundtrip; integration: two processes discover each other (documented).

### Milestone 3 ‚Äî Peer state machine
- `peer`: `PeerState { Disconnected, Discovering, Connecting, Handshaking, Connected, Dead }`, `Peer { id, addrs, state, last_seen, session }`, `PeerManager` (DashMap-backed), `transition()` with illegal-transition errors, `PeerEvent` channel.
- **Verification**: unit tests for every legal/illegal transition, manager add/remove/lookup.

### Milestone 4 ‚Äî Noise XX handshake
- `crypto`: `NoiseBuilder::new_responder/initiator(network_secret, device_keys)` using `snow::Builder::new("Noise_XX_25519_ChaChaPoly_BLAKE2s".parse())` with `.prologue(&network_secret)`, `.local_private_key(&x25519)`, `.remote_public_key(...)`.
- `HandshakeState` driver for the 3-message XX exchange; `TransportState` after success ‚Üí `encrypt`/`decrypt` with ChaCha20-Poly1305.
- Mutual auth: prologue mismatch ‚Üí handshake fails (peer not in network).
- **Verification**: two local `TransportState`s complete XX, encrypt/decrypt roundtrip, wrong-prologue test fails. Pure unit tests (no network).

### Milestone 5 ‚Äî Reliable message layer
- `peer`: frame protocol `[u32 BE len][payload]` over UDP (or optional QUINN), `Message { Data, Heartbeat, Ping, Pong, SessionInit }`, heartbeat task (every 15s), reconnect with backoff, session expiration (60s no traffic ‚Üí Dead), MTU fragmentation (>1280 ‚Üí split + reassemble).
- `MessageChannel`: `send`/`recv` over a `UdpSocket`, bounded channels to engine.
- **Verification**: heartbeat timer fires, session expires on timeout, fragmentation reassembly roundtrip unit test.

### Milestone 6 ‚Äî Cross-platform TUN
- `tun`: `create_tun(overlay_ip, mtu) -> AsyncDevice` using `tun::Configuration`, platform-specific config (macOS utun, Linux tunl, Windows wintun). Read/write loops via `tokio::io::AsyncReadExt`/`AsyncWriteExt`.
- Document privilege needs (macOS: root or `tun` entitlement; Linux: root/cap_net_admin; Windows: admin + wintun.dll).
- **Verification**: `cargo build` on all 3 targets; manual `cargo run` creates interface (needs privileges, documented). Unit test: configuration builder.

### Milestone 7 ‚Äî Overlay IP allocation
- `overlay`: `overlay_ip(pubkey) -> Ipv4Addr` deterministic in 10.88.0.0/16, `claim_ip(dht, ip, pubkey)` via DHT immutable-store put, `resolve_ip(dht, ip) -> Option<PubKey>` collision check. On collision ‚Üí fallback to next derived slot.
- **Verification**: deterministic across runs; 1000 distinct pubkeys ‚Üí no collisions in test; collision-resolution unit test.

### Milestone 8 ‚Äî Routing
- `routing`: `Router { map: OverlayIp ‚Üí PeerId }`, parse IPv4 packet header (`etherparse` or hand-rolled), dst-IP lookup, forward: `encrypt(packet) ‚Üí peer.send()`; inbound: `recv ‚Üí decrypt ‚Üí tun.write()`.
- **Verification**: unit tests for packet parse + route-table lookup + encrypt/decrypt-through-router roundtrip.

### Milestone 9 ‚Äî Peer management orchestration
- `core`: `SeedNet::run(seed)` orchestrates DHT announce loop, discovery loop, peer connect (state machine + Noise + message layer), keepalive, NAT rebinding (re-announce on addr change), peer timeout pruning. `down`/`status` via PID + state file.
- **Verification**: `cargo run -- up "<seed>"` brings up TUN, joins DHT, connects to discovered peers, logs state transitions. `status` prints running state.

### Milestone 10 ‚Äî Cross-platform integration test
- End-to-end two-process test harness (script + Rust test): two SeedNet instances on loopback, distinct device identities, same seed ‚Üí ping overlay IPs succeeds.
- Cross-platform build matrix verification (Linux/macOS via CI config, Windows documented).
- **Verification**: `ping 10.88.x.y` from device A ‚Üí B over the encrypted overlay; tcpdump shows only encrypted UDP on the wire.

## Key technical choices (locked)
- **Edition 2024**, resolver 3, MSRV pinned to 1.93.1 via `rust-toolchain.toml`.
- `mainline` 7.0.0 (async feature) for DHT ‚Äî documented substitution for non-existent `bittorrent-dht`.
- `snow` 0.10.0, pattern `Noise_XX_25519_ChaChaPoly_BLAKE2s`.
- `tun` 0.8.13 (`async`+`tokio` features).
- `dashmap` 6.1 (stable; avoid 7.0.0-rc2).
- `tokio` 1.52 full features, `tracing`/`tracing-subscriber` for structured logs, `thiserror` for error enums, `clap` v4 derive.
- UDP-based encrypted transport for data; QUINN kept as optional/future (per spec's "(optional)").
- No `unsafe` in our code; all `unsafe` confined to vetted dependencies.
